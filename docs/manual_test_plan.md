# 全機能確認チェックリスト (ローカル動作確認用)

以下の手順で、ローカル環境 (`npx opennextjs-cloudflare preview`) 上での全機能の動作確認を行ってください。
※ 事前に `npm run build:cf` を実行済みであることを前提とします。

## 0. サーバー起動
- [ ] ターミナルで `npx opennextjs-cloudflare preview` を実行。
- [ ] 指定されたURL (例: `http://localhost:8787`) にブラウザでアクセス。

## 1. 認証機能 (Authentication)
- [ ] **新規登録 (学生)**:
    - [ ] `kamiyama.ac.jp` ドメインのメールアドレスで登録できること。
    - [ ] `gmail.com` 等、他ドメインでの登録がエラーになること (ドメイン制限の確認)。
- [ ] **ログイン**:
    - [ ] 登録したアカウントでログインできること。
- [ ] **ログアウト**:
    - [ ] サイドバーまたはヘッダーからログアウトできること。

## 2. 学生向け機能 (Student)
ユーザー: 学生アカウントでログイン

### プロフィール (`/student/profile`)
- [ ] プロフィール情報の表示確認。
- [ ] 編集機能が動作し、保存されること。

### 求人・インターンシップ (`/student/jobs`)
- [ ] 求人一覧が表示されること。
- [ ] 検索・フィルタリング機能があれば動作すること。
- [ ] 個別の求人詳細ページ (`/student/jobs/[id]`) が開けること。
- [ ] 「応募する」ボタンから応募が完了すること。

### 応募履歴 (`/student/applications`)
- [ ] 応募した求人が一覧に表示されていること。
- [ ] ステータス（選考中、マッチング済み等）が確認できること。

### マッチング後 (`/student/intern/[id]`)
- [ ] (企業側で承諾後) マッチング詳細画面にアクセスできること。

## 3. 企業向け機能 (Company)
ユーザー: 企業アカウントでログイン

### プロフィール (`/company/profile`)
- [ ] 企業情報の表示・編集・保存ができること。

### 求人管理 (`/company/jobs`)
- [ ] 作成済みの求人一覧が表示されること。
- [ ] **新規作成** (`/company/jobs/new`): 求人を作成し、公開できること。
- [ ] **編集**: 既存の求人内容を変更し、保存できること。

### 応募者管理 (`/company/applicants`)
- [ ] 学生からの応募が一覧に表示されていること。
- [ ] 応募者の詳細プロフィールが閲覧できること。
- [ ] **選考アクション**: 「採用(マッチング)」または「不採用」のステータス変更ができること。

### インターンシップ管理 (`/company/intern/[id]`)
- [ ] マッチングした学生との詳細画面（実習情報など）が表示されること。

## 4. 管理者機能 (Admin)
ユーザー: 管理者権限を持つアカウント

- [ ] **ダッシュボード** (`/admin/dashboard`): 全体の統計情報確認。
- [ ] **ユーザー管理** (`/admin/users`): 登録ユーザー一覧の確認。
- [ ] **企業管理** (`/admin/companies`): 登録企業一覧と承認ステータスの変更。
- [ ] **求人管理** (`/admin/jobs`): 全求人の確認。
- [ ] **応募管理** (`/admin/applications`): 全応募履歴の確認。

## 5. その他
- [ ] 全ページにおいて、ヘッダー/フッター/サイドバーのリンクが正しく機能していること。
- [ ] 存在しないページにアクセスした際、404エラーページが表示されること。
